Track Installs and Purchases Without a Backend

iOS Development · April 2026

No backend, no SDK, no API key management. In this guide I'll show you how to get an instant notification on your phone every time someone downloads your app or makes a purchase — using nothing but a simple HTTP request.

What is ntfy?

ntfy (pronounced "notify") is a free, open-source HTTP-based notification service. You send a POST request to a URL, and your phone gets a notification instantly.

Why ntfy?

• No SDK — standard URLSession is all you need
• No user data is transmitted
• Runs completely invisibly inside your app
• Free, no account required
• Compliant with Apple App Store guidelines (anonymous, no personal data)
ntfy works on the format ntfy.sh/TOPIC_NAME. Your topic name is your "password" — choose something that can't be guessed.

Setup

1
Download the ntfy app
Search for "ntfy" on the App Store and download the free app.

Once open, tap "+" and subscribe to your topics. Example topic names:

• myapp-install-x7k2p9 (for install notifications)
• myapp-premium-n5x8j4 (for purchase notifications)

Make topic names hard to guess — use a UUID or random characters.
Topic names are public. If yours can be easily guessed, anyone can subscribe. Don't use simple names like "myapp-installs".
2
Create NtfyService.swift
Add a new Swift file to your project:
import Foundation // MARK: - Ntfy Service actor NtfyService { static let shared = NtfyService() private init() {} private let installTopic = "myapp-install-x7k2p9" private let premiumTopic = "myapp-premium-n5x8j4" func onboardingStarted() async { await send(topic: installTopic, message: "New user") } func premiumPurchased() async { await send(topic: premiumTopic, message: "New purchase") } private func send(topic: String, message: String) async { guard let url = URL(string: "https://ntfy.sh/\(topic)") else { return } var request = URLRequest(url: url) request.httpMethod = "POST" request.httpBody = message.data(using: .utf8) request.timeoutInterval = 5 do { let (_, response) = try await URLSession.shared.data(for: request) #if DEBUG let status = (response as? HTTPURLResponse)?.statusCode ?? 0 print("[Ntfy] \(topic) → \(status)") #endif } catch { // Fail silently — never affects the user } } }
Using actor provides Swift 6 compatibility and guarantees thread safety — no NSLock or DispatchQueue needed.
3
Add to onboarding
Add this where the user taps the first "Continue" button:
// Fire on the transition from page 0 → 1 if currentPage == 0 { Task { await NtfyService.shared.onboardingStarted() } } // Normal page transition continues currentPage += 1
Trigger on the first page, not the last. This way you count the install even if the user abandons onboarding halfway through.
4
Add to purchases
With StoreKit 2, all purchases flow through Transaction.updates. Adding the notification here catches normal purchases, promo codes, and restores:
private func listenForTransactions() { Task.detached { for await result in Transaction.updates { if case .verified(let transaction) = result { await transaction.finish() await self.checkEntitlements() // Ntfy notification Task { await NtfyService.shared.premiumPurchased() } } } } }
Add to Transaction.updates, not purchase(). This catches all transactions that come through the App Store — including those made while the app was closed.

Testing

Test in the Simulator
Run the app in Simulator, go through onboarding and tap "Continue" on the first page. You should see this in the Xcode console:

[Ntfy] myapp-install-x7k2p9 → 200

At the same moment, a notification should arrive on your phone in the ntfy app.

Got 200 but no notification?
• Check the topic name in the ntfy app
• Check notification permissions
• Unsubscribe from the channel and re-subscribe

App Store compliance

Is this approach compliant with Apple's guidelines?
Yes — as long as these conditions are met:

• No personal data is sent: You're sending a fixed string like "New user" — no device ID, IP address, or user information
• The user doesn't notice: It runs silently in the background and has no effect on the user experience
• It's your own notification, not analytics: Data isn't shared with third parties

Apple Guideline 5.1.2 restricts tracking user data. Sending an anonymous signal doesn't fall under this restriction.
You don't need to declare this HTTP request in PrivacyInfo.xcprivacy — no user data is being processed.

Limits

How much can you use on the free plan?
The ntfy.sh free plan allows a burst of 60 messages, then replenishes at one message every 5 seconds. That translates to hundreds of installs per day — more than enough for most indie apps.

If you hit the limit: The app fails silently. The user is not affected. You just miss that one notification.

If you need higher limits, ntfy.sh has a paid plan (~$5/month), or you can self-host ntfy on your own server.
